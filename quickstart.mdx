---
title: "Quick Start"
description: "Install the CLI, record a site, and make your first API call in under 5 minutes."
---

## 1. Install the CLI

```bash
npm install -g @browserwire/cli
```

This installs the `bw` command globally. Playwright and Chromium are installed automatically via the `postinstall` script.

<Note>Requires **Node.js 22+** and **macOS or Linux**.</Note>

## 2. Log in

```bash
bw login
```

Your browser opens for authentication. Once you sign in, a session token is saved locally at `~/.config/browserwire/config.json`.

## 3. Record a site

```bash
bw record https://example.com
```

A Chrome browser opens. Navigate the site freely — the recording hasn't started yet.

When you're ready, **press Enter** to start recording. Interact with the site: fill forms, click buttons, navigate pages. When you're done, **press Enter** again to stop.

The recording is uploaded automatically, and training begins in the cloud. Your dashboard opens so you can monitor progress.

## 4. Wait for training

Training typically takes 1–2 minutes. You can monitor progress on the [dashboard](https://app.browserwire.io/dashboard) — look for the training status badge next to your recording.

Once complete, a **manifest** appears in the APIs section of the dashboard with every view and action the AI discovered.

## 5. Start the local server

```bash
bw run
```

```
BrowserWire execution server running on http://127.0.0.1:8787
  Health:    curl http://127.0.0.1:8787/api/health
  Sites:     curl http://127.0.0.1:8787/api/sites
  Headed mode | Max concurrent: 3
```

A browser launches in the background. The server pre-fetches your manifests and is ready to serve requests.

## 6. Call your APIs

List discovered sites:

```bash
curl http://127.0.0.1:8787/api/sites
```

Read a view (structured data from a page):

```bash
curl http://127.0.0.1:8787/api/sites/example-com/views/HomePage/product_list
```

Execute an action:

```bash
curl -X POST http://127.0.0.1:8787/api/sites/example-com/actions/LoginPage/submit_login \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "secret"}'
```

<Tip>
  The slug (`example-com`) is the site's domain with dots replaced by hyphens. Check `GET /api/sites` to see all available slugs.
</Tip>
